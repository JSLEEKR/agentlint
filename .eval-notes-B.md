## Bugs Found

### BUG-1 (HIGH): CRLF line endings break markdown parsing completely
- **File**: `src/scanner.ts:75`
- **Description**: Files with `\r\n` (CRLF) line endings -- standard on Windows -- caused the markdown parser to miss ALL headings, code blocks, links, and file references. `content.split("\n")` produces lines with trailing `\r`. The heading regex `^(#{1,6})\s+(.+)$` and code fence regex `^(\`{3,})([\w-]*)$` both use `$` which does not match before `\r`. On a cross-platform linter, this is a critical bug -- any CLAUDE.md saved on Windows would produce zero structural warnings (false "clean") while also falsely triggering "no headings" warnings.
- **Root cause**: No CRLF normalization in the scanner's line-splitting logic.
- **Fix**: Added `.map((l) => l.replace(/\r$/, ""))` after `content.split("\n")` in `scanFiles()` to strip trailing `\r` from all lines before passing to parsers.
- **Regression tests**: +3 parser tests (headings, code blocks, links with CRLF), +1 scanner test (CRLF line stripping), +2 integration tests (CRLF CLAUDE.md end-to-end, CRLF secret detection)

### BUG-2 (MEDIUM): dist/ not rebuilt after Cycle A source changes
- **File**: `dist/engine.js` (compiled output)
- **Description**: Cycle A fixed `src/engine.ts` to use unconditional config severity (`severity,` instead of `diag.severity ?? severity`), but never ran `tsc` to rebuild the dist/ directory. The compiled `dist/engine.js` still had the old code `diag.severity ?? severity`, meaning Cycle A's BUG-1 fix (config severity override) was never actually applied at runtime. Rules that always provide their own severity in `report()` calls would always use the rule's severity, ignoring the user's config setting.
- **Root cause**: Source change not followed by rebuild.
- **Fix**: Ran `npx tsc` to rebuild dist/ from source. Verified dist/engine.js now has `severity,` (unconditional config override).
- **Regression test**: +1 engine test ("should override rule-internal severity with config severity for all diagnostics") that verifies struct-claude-md-sections with config "error" overrides the rule's internal "info" for long-section diagnostics.
