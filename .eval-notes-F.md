11. `bash -c "echo"` extracts `-c` as script path -- minor heuristic limitation.

14. `void md;` statements in rules -- TypeScript linting suppression for noUnusedLocals (void expression satisfies the checker without behavioral effect).
15. `_hookName` unused variable pattern -- intentional destructuring with underscore prefix convention.
16. Double detection of Anthropic key matching both generic and specific patterns -- correct, two distinct regex patterns match independently.
17. Config in parent directory not searched -- documented behavior (project root only).
18. No warning for unrecognized config keys -- standard JSON config convention.
19. `parseFrontmatterYaml` accepts null as empty object -- intentional YAML null handling.
20. `--fix` output shows diagnostics even after fixing -- standard ESLint behavior (shows what was fixed).
21. Heading hierarchy rule flags starting at h2 as first heading -- debatable but consistent behavior.
22. Standalone `o1`/`o3` model names not matched by ref-model-valid regex -- intentional to avoid false positives on common text like "option o1" or "item o3". The regex requires a suffix like `-mini` or `-preview`.
